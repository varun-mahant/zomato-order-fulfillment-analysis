# Zomato Order Fulfillment Analysis

## 📊 Project Overview
This project analyzes 42,000+ Zomato order records to understand delivery efficiency, identify delays, and evaluate customer satisfaction patterns.  
A strong focus was placed on **data cleaning and transformation** to ensure accurate and reliable analysis.

## 🎯 Objectives
- Analyze delivery performance across different time periods  
- Identify peak delay periods and operational bottlenecks  
- Understand customer satisfaction trends  
- Explore order distribution by city type and vehicle type  

## 🛠 Tools & Technologies
- Microsoft Excel  
- Power Query (ETL)  
- Pivot Tables & Charts  
- Data Visualization  

## 🧹 Data Cleaning & Transformation (Key Highlight)
- Handled missing and inconsistent values  
- Standardized categorical fields (city type, ratings, etc.)  
- Created derived columns (daypart, processing time, delivery speed)  
- Cleaned timestamp data for accurate time-based analysis  
- Built a structured ETL pipeline using Power Query  

## 📈 Key Insights
- Metropolitan cities contribute ~75% of total orders  
- Morning shows the highest fast delivery rate  
- Evening orders face the highest delays (peak-hour pressure)  
- Motorcycles dominate deliveries (~58%), indicating preference for speed  

## 📊 Dashboard
An interactive Excel dashboard was built using slicers and pivot charts to visualize:
- Weekly order trends  
- Delivery speed by time of day  
- Order distribution by city and vehicle type  
- Customer ratings distribution  

## 📂 Dataset
- 42,000+ order-level records  
- Includes timestamps, delivery duration, ratings, and external factors  

## 🚀 Outcome
Cleaned and transformed raw data into a structured format, enabling accurate analysis and actionable insights for improving delivery performance and customer experience.
